Red-Black Trees

A Red-Black tree is a data structure based on a binary tree strcutre.

Each operation takes worst case O(log⁡n)O(\log n)O(logn) time.

  • The root is black
  • All root-to-leaf paths have the same number of black nodes
  • Red nodes cannot have red children

Any 2-3-4 tree can be converted into a Red-Black tree (and vice-versa)

Building Multi-level Linked Lists by Flipping Coins

Flip one coin for each key:

  • for each key that got a head, put it in the new top level
  • repeat with the keys from the new top level
    • stop when the top level contains only the smallest and largest keys

The intuition is that nnn coin flips contains about n2\frac{n}{2}2n​ heads and about n2\frac{n}{2}2n​ tails - the heads are roughly evenly spread out.

Skip Lists

A skip list is a mult-level linked list where the INSERTINSERTINSERTs are done by flipping coins.

INSERT(x,k):INSERT(x,k):INSERT(x,k):

  1. Use FIND(k)FIND(k)FIND(k) to insert (x,k)(x,k)(x,k) into the bottom level
  2. Flip a coin repeatedly:
    • If you get a heads, insert (x,k)(x,k)(x,k) into the next level up
    • If you get a tails, stop

DELETE(k):DELETE(k):DELETE(k):

  1. Use FIND(k)FIND(k)FIND(k) to find (x,k)(x,k)(x,k)
  2. Delete (x,k)(x,k)(x,k) from all levels
  3. Remove any empty levels (containing only the smallest and largest keys)

Each of these operations take expected O(log⁡n)O(\log n)O(logn) time.

Why?

After nnn INSERTINSERTINSERT operations, the probability that a skip list has more than 2log⁡n2\log n2logn levels is at most:

1n\dfrac{1}{n} n1​

As the number of levels is O(log⁡n)O(\log n)O(logn) (with high probability), we can conclude the number of times we move down is very unlikely to be O(log⁡n)O(\log n)O(logn).
