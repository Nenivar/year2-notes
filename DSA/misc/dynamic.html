Dynamic Programming

Dynamic programming is a technique for finding efficient algorithms for problems which can be broken down into simpler, overlapping subproblems.

Basic idea:

  1. Find a recursive formula for the problem, in terms of answers to subproblems
  2. Write down a naive recursive algorithm
  3. Speed it up by storing the solutions to subproblems
  4. Derive an iterative algorithm by solving the subproblems in a good order

Dynamic Progamming is recursion without repetition.

Largest Empty Square

Given an n×nn\times nn×n monochrome image, find the largest empty square.

Find a Recursive Formula

Any m×mm\times mm×m square of pixels, SSS is empty iff

  • The bottom-right pixel of SSS is empty
  • The three (m−1)×(m−1)(m-1)\times(m-1)(m−1)×(m−1) squares in the top-left, > top-right and bottom-left of SSS are empty

Let LES(x,y)LES(x,y)LES(x,y) be the size of the largest empty square whose bottom right is at (x,y)(x,y)(x,y)

Then:

  • If the pixel (x,y)(x,y)(x,y) is not empty, then LES(x,y)=0LES(x,y)=0LES(x,y)=0

  • If (x,y)(x,y)(x,y) is empty and in the first row or column, LES(x,y)=1LES(x,y)=1LES(x,y)=1

  • If (x,y)(x,y)(x,y) is empty and not in the first row or column,

    LES(x,y)=min(LES(x−1,y−1),LES(x−1,y),LES(x,y−1))+1LES(x,y)=min(LES(x-1,y-1),LES(x-1,y),LES(x,y-1))+1LES(x,y)=min(LES(x−1,y−1),LES(x−1,y),LES(x,y−1))+1

Write Down a Recursive Algorithm

We can use the recursive formula to get a recusrive algorithm:

def LES(x,y):
    if pixel (x,y) is not empty:
        return 0
    if x == 1 or y == 1
        return 1
    return min(LES(x-1,y-1), 
               LES(x-1, y),
               LES(x,y-1)) + 1

This computes the size of the largest empty square whose bottom right is at (x,y)(x,y)(x,y).

∴\therefore∴ the maximum of LES(x,y)LES(x,y)LES(x,y) over all xxx and yyy gives the size of the largest empty square in the whole image.

Efficiency

The running time of LES(n,n)LES(n,n)LES(n,n) is currently exponential in nnn.

Store the Solutions to Subproblems

def MEMLES(x,y):
    if pixel (x,y) is not empty:
        return 0
    if x == 1 or y == 1
        return 1
    if LES[x,y] undefined:
        LES[x,y] = min(LES(x-1,y-1), 
                       LES(x-1, y),
                       LES(x,y-1)) + 1
    return LES[x,y]

In this version of the algorithm, we store solutions ot previously computed subproblems in an (n×n)(n\times n)(n×n) 2D array called LES.

This is called memoization.

Now, each entry LES[x,y] is only computed once.

The time complexity of computing MEMLES(n,n) is now O(n2)O(n^2)O(n2)

To use this to get an iterative algorithm, fill in the array from the top-left.

Derive an Iterative Algorithm

def ITLES(n):
    for y in range(1,n):
        for x in range(1,n):
            if pixel (x,y) is not empty:
                LES[x,y] = 0
            elif x==1 or y==1:
                LES[x,y] = 1
            else:
                LES[x,y] = min(LES(x-1,y-1), 
                               LES(x-1, y),
                               LES(x,y-1)) + 1

This iterative version of the algorithm runs in O(n2)O(n^2)O(n2) time, and avoids making any recursive calls.

Weighted Interval Scheduling

Given nnn weighted intervals, find the schedule with largest total weight.

  • Two intervals are compatible if they don't overlap
  • A schedule is a set of compatible intervals
  • The weight of a schedule is the sum of the weight of the intervals it contains

The intervals are given in an array AAA of length nnn.

A[i]A[i]A[i] stores a triple (si,fi,wi)(s_i,f_i,w_i)(si​,fi​,wi​) which defines the ithi^{th}ith interval.

Let p(i)p(i)p(i) be the rightmost interval (in order of finish time) which finishes before the ithi^{th}ith interval but doesn't overlap it. If no such interval exists, p(i)=0p(i)=0p(i)=0

def WIS(i):

Store The Solutions to Subproblems

def WIS(i):

        return 0

    return WIS[i]

  • Each entry WIS[i] is only computed once
  • The time complexity of computing MEMWIS(n) is now O(n)O(n)O(n), because every recursion causes an unfilled entry to be filled in the array

To compute WIS[i] we need WIS[i-1] and WIS[p(i)], both of which are to the left of WIS[i].

Derive an Iterative Algorithm

def ITWIS(n):
    if i==0:
        return 0
    for i in range(1,n):
        WIS[i] = max(WIS[i-1], WIS[p(i)] + w_i)
    return WIS[i]

The Actual Algorithm

ITWIS(n) finds the weight of the optimal schedule, but doesn't find the actual schedule.

def FINDWIS(i):
    if i==0:
        return None
    if WIS[i-1] <= WIS[p(i)] + w_i:
        return FINDWIS(p(i)) then i
    return FINDWIS(i-1)

This is called backtracking.

  1. Find all the p(i)p(i)p(i) values - O(nlog⁡n)O(n\log n)O(nlogn)
  2. Run ITWIS(n) to find the optimal weight - O(n)O(n)O(n)
  3. Run FINDWIS(n) to find the schedule - O(n)O(n)O(n)

Overall this takes O(nlog⁡n)O(n\log n)O(nlogn) time.
