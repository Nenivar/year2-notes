P & NP

  • Decision Problems - problems with a yes/no answer
  • PPP - set of decision problem families with running time polynomial in input size

Reduction

Determine which families of problems are in PPP - we can determine this for some families using reductions.

A reduction maps family AAA to family BBB in polynomial time s.t:

  • the answer for an instance of AAA is 'yes' ⇔\Leftrightarrow⇔ answer for its image in BBB is 'yes'

If we have an algorithm with runtime polynomial in input size that, given any instance of decision problem family AAA, transforms it into an instance of decision problem family BBB s.t:

  • answer for the instance of AAA is yes ⇔\Leftrightarrow⇔ answer for instance of BBB is yes

Then say: there is a polynomial-time reduction of AAA to BBB (or: AAA reduces to BBB)

  • (or: A≤pBA \leq_p BA≤p​B)

If A≤pBA\leq_p BA≤p​B, and we can solve BBB, we can use this to solve AAA.

Properties

  • If AAA is in PPP and B≤pAB\leq_p AB≤p​A, then BBB is in PPP.
  • For any decision problem family AAA, there is a polynomial-time reduction of AAA to AAA; A≤pAA\leq_p AA≤p​A
  • If A≤pBA\leq_p BA≤p​B and B≤pCB\leq_p CB≤p​C then A≤pCA\leq_p CA≤p​C

Example

  • INDEPENDENT_SET: is there a set of kkk vertices in GGG s.t. no two vertices in the set are linked by an edge?
  • CLIQUE: is there a set of kkk vertices in GGG s.t. every pair of vertices in the set is linked by an edge?

INDEPENDENT_SET[G,k] has the answer 'yes' iff CLIQUE[G',k] has the answer 'yes'.

Verifying Solutions

NPNPNP is the class of problems for which, if the answer to an instance is 'yes', we can verify this efficiently.

NPNPNP stands for 'Non-deterministic polynomial'

Not every decision problem family is in NPNPNP.

For example: given a position in an n×nn\times nn×n chess, can White force a win?

AAA is in PPP implies AAA is in NPNPNP

  • We can efficiently verify a claim that an instance of AAA has answer 'yes', by solving AAA ourselves

∴P⊆NP\therefore P\subseteq NP ∴P⊆NP

NPNPNP-hard

For every BBB in NPNPNP, B≤pAB\leq_p AB≤p​A

(AAA is at least as hard as the hardest problems in NPNPNP)

  • If there's a polynomial-time algorithm for a problem family in NPNPNP-hard, then P=NPP=NPP=NP
  • So if we can prove a problem family is in NPNPNP-hard, this is evidence there's no polynomial-time algorithm to solve it

A is in NPNPNP-hard doesn't necessarily imply AAA is in NPNPNP.

e.g. the n×nn\times nn×n chess problem family mentioned earlier is in NPNPNP-hard but not in NPNPNP

NP−completeNP-completeNP−complete

AAA is in NPNPNP-hard and AAA is in NPNPNP

If you can find a polynomial-time algorithm to solve just one of these families:

P=NPP=NP P=NP

If you can prove that for just one NP-complete decision problem family, there's no efficient algorithm to solve it:

P≠NPP=\not NP P≠​NP

Proving NP-Completeness by a Reduction

Suppose AAA is in NPNPNP-complete, BBB is in NPNPNP, and you can show that A≤pBA\leq_p BA≤p​B:

  • since AAA is in NPNPNP-hard, for every CCC in NPNPNP:

    C≤pAC\leq_p A C≤p​A

  • for every CCC in NPNPNP, we have C≤pAC\leq_p AC≤p​A and A≤BA\leq BA≤B, so for every CCC in NPNPNP:

    C≤pBC\leq_p B C≤p​B

    (i.e. BBB is in NPNPNP-hard)

  • and since BBB is in NPNPNP, it follows that BBB is in NPNPNP-complete

AAA in NPNPNP-complete reduces to BBB in NP⇒BNP \Rightarrow BNP⇒B is in NPNPNP complete
